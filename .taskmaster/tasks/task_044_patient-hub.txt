## 1. Design and Implement Document Upload Form [pending]
### Dependencies: None

### Details:

## 2. Set Up HTMX for Asynchronous Document Upload [pending]
### Dependencies: 44.1
### Description: Configure the document upload form to submit asynchronously using HTMX, preventing page reloads and enhancing user experience.
### Details:
Add the 'hx-post' attribute to the form element, pointing to the appropriate endpoint for document upload. Set 'hx-target' to specify where the response should be rendered, and 'hx-swap' to define how the content should be swapped. Ensure the form's 'enctype' is set to 'multipart/form-data' to handle file uploads. Refer to HTMX documentation for detailed implementation guidance. ([four.htmx.org](https://four.htmx.org/docs/?utm_source=openai))

## 3. Implement Document Upload Endpoint in Django [pending]
### Dependencies: 44.2
### Description: Create a Django view to handle the document upload request, process the uploaded file, and return the updated document row partial.
### Details:
Define a view function that accepts POST requests at the '/patients/<pk>/upload-document/' endpoint. In this view, handle the uploaded file, perform necessary validations, and save the document. After processing, render the '_document_row.html' partial with the new document information and return it as the response. Ensure that the view handles errors gracefully and returns appropriate error messages when validation fails. ([docs.djangoproject.com](https://docs.djangoproject.com/en/3.2/topics/http/file-uploads/?utm_source=openai))
