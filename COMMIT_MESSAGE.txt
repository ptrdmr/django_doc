feat(documents): implement verification actions for flagged documents (Task 41.26)

Completed implementation of three verification action handlers for the optimistic
concurrency merge system, enabling medical staff to review and resolve flagged
documents through Mark as Correct, Correct Data, and Rollback Merge workflows.

## Features Implemented

### 1. Mark as Correct Action
- View function with comprehensive error handling
- Approves flagged data without changes, sets review_status='reviewed'
- HIPAA-compliant audit trail (reviewer, timestamp, notes)
- User-friendly success/error messages
- POST-only endpoint with authentication required

### 2. Correct Data Action  
- GET/POST view with form pre-population
- CorrectDataForm with JSON structure validation
- Professional UI template with help text and tips
- Validates FHIR resourceType fields
- Review notes field for audit documentation
- Atomic save operation (FHIR data + approval status)

### 3. Rollback Merge Action
- Reverses optimistic merge by removing FHIR resources from patient record
- Atomic transaction ensures data integrity (all-or-nothing)
- Resets ParsedData status to 'pending', clears merge flags
- Logs rollback action in flag_reason for audit trail
- Confirmation dialog prevents accidental rollbacks

## Known Limitation & Future Work
The "Correct Data" action currently uses raw FHIR JSON editing, which is not
ideal for medical professionals. This is a functional placeholder that serves
as a power-user fallback.

**Future Enhancement**: Comprehensive PRD created for guided wizard interface
(.taskmaster/docs/guided-wizard-correction-prd.md) that will:
- Present data as editable form fields (not raw JSON)
- Show one issue at a time with clinical context
- Provide AI-assisted suggestions (e.g., "Did you mean Metformin 500mg BID?")
- Display OCR preview with highlighted source text
- Integrate with RxNorm/ICD-10 APIs for validation
- Estimated 8-12 development days for full implementation

Primary workflows remain "Mark as Correct" (most common) and "Rollback Merge"
(for errors). Raw JSON editing serves edge cases requiring advanced control.
